Résumé du cours

Dans Angular, il existe deux approches principales pour la création de formulaires :

Formulaire basé sur un modèle (Template-Driven) :
Dans cette approche, la logique du formulaire est principalement définie dans le fichier de modèle HTML. Il faut spécifier les règles de validation directement dans le modèle et Angular infère la structure du formulaire à partir de celui-ci.
Formulaire basé sur les données (Data-Driven) :
Ici, la logique du formulaire est principalement définie dans le composant TypeScript. Il faut explicitement créer les contrôles de formulaire dans le code TypeScript et définir les règles de validation et comportements associés.

Le FormGroup est un des fondamentaux de la construction de formulaires sur Angular. Il regroupe plusieurs contrôles de formulaire tels que les FormControl et les FormArray. De plus, il peut également contenir d'autres FormGroup, permettant une organisation hiérarchique des données du formulaire. 

        template HTML
        <form (ngSubmit)="submitForm()">
          ... Le contenu du formulaire ici ...
        </form>
      
      component TS

Conclusion

(ngSubmit) :
C'est un événement déclenché lors de la soumission du formulaire.
Il faut lier cette action à une fonction dans le composant TypeScript.
submitForm() :
C'est la fonction à exécuter lorsque le formulaire est soumis.
Pour gérer la logique de soumission, il faut définir dans le composant.
En utilisant ces concepts, je peux créer des formulaires efficaces et bien organisés pour mes futures applications Angular.
